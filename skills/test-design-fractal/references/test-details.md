# 测试设计详细参考

## 文档模板结构

```markdown
# 分形式测试用例设计 - {项目名称} - {YYYYMMDD}
## 任务信息（开始时间、项目名、测试目标）
# L0-L4 各层级嵌套内容（L2按测试类型拆分，L4为具体用例）
# 总结与验证 + 测试用例清单
```

**保存路径**：`docs/test/test-design-fractal-{YYYYMMDD}.md`

## L2 测试类型详细分类

| 类型 | 覆盖重点 | 典型场景 |
|------|----------|----------|
| 功能测试 | 业务逻辑正确性 | 正常流程、核心功能 |
| 性能测试 | 响应时间/吞吐量 | 并发、大数据量 |
| 安全测试 | 漏洞防护 | 注入、XSS、越权 |
| 边界测试 | 极值处理 | 空值、超长、负数 |
| 异常测试 | 错误处理 | 断网、超时、格式错误 |
| 回归测试 | 变更无副作用 | 重构后全功能 |

## L4 测试用例标准格式

每条L4用例包含：

| 字段 | 说明 |
|------|------|
| 用例ID | TC-{模块}-{序号} |
| 用例名称 | 简明描述 |
| 前置条件 | 执行前需满足的状态 |
| 测试步骤 | 操作序列(1/2/3...) |
| 测试数据 | 输入数据 |
| 预期结果 | 期望输出 |
| 优先级 | P0(阻塞)/P1(重要)/P2(一般) |
| 追溯需求 | 关联的REQ-ID或用户故事 |

## 决策点完整清单

| 层级 | 必须确认的决策点 |
|------|------------------|
| L0 | 测试目标、范围、通过标准 |
| L1 | 功能模块优先级、覆盖策略 |
| L2 | 测试类型选择、每种类型的深度 |
| L3 | 场景完整性、边界值选取 |
| L4 | 用例步骤细化程度、数据准备方案 |
