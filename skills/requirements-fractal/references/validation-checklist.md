# 需求分析验证清单

## 验证时机

| 验证阶段 | 时机 | 内容 |
|----------|------|------|
| 层级内验证 | 每个层级完成后 | 当前层级的正确性和完整性 |
| 层间验证 | 进入下一层级前 | 上下层级的一致性 |
| 最终验证 | 全部分析完成后 | 正向+反向+一致性全面检查 |

## 各阶段验证清单

### L0 完成后验证

- [ ] **目标清晰性**：需求目标是否明确可理解？
- [ ] **价值确认**：为什么做这个需求？业务价值是什么？
- [ ] **范围界定**：MVP范围和完整范围的边界是否明确？
- [ ] **决策记录**：L0所有决策点是否已记录用户选择？
- [ ] **时间戳**：L0完成时间是否已记录？

### L1 完成后验证

- [ ] **模块完整性**：功能模块划分是否覆盖全部需求范围？
- [ ] **模块独立性**：各模块职责是否清晰、无重叠？
- [ ] **优先级合理**：模块优先级排序是否有依据？
- [ ] **横向拆分确认**：L1拆分方案已通过 AskUserQuestion 确认？
- [ ] **决策追溯**：每个模块的决策是否追溯到 L0 目标？

### L2 完成后验证

- [ ] **子功能覆盖**：子功能是否完整覆盖父模块？
- [ ] **依赖关系**：子功能间的依赖关系是否明确？
- [ ] **用户故事对应**：每个子功能是否关联用户故事？

### L3 完成后验证

- [ ] **需求可测试**：每条需求是否有明确的验收标准？
- [ ] **需求唯一ID**：每条需求是否分配了 REQ-ID？
- [ ] **非功能需求**：性能/安全/可用性等非功能需求是否覆盖？

## 最终验证（任务完成后）

### 正向验证（需求 → 文档）

逐条检查每条需求，确保：

- [ ] 每条 L3/L4 需求在文档中有详细描述
- [ ] 每条需求的验收标准已明确定义
- [ ] 每条需求的优先级已标注
- [ ] 每条需求可追溯到 L0 目标

### 反向验证（文档 → 需求）

遍历所有文档内容，确保：

- [ ] 文档中的每个功能描述都有对应的需求项
- [ ] 文档中的每个决策都有对应的决策记录
- [ ] 无"悬空"的内容（有描述但无需求支撑）
- [ ] 无遗漏的重要需求

### 一致性验证

- [ ] **L0 → L1**：L0的目标是否被 L1 的所有模块覆盖？
- [ ] **L1 → L2**：L1 模块的职责是否与 L2 子功能匹配？
- [ ] **L2 → L3**：L2 子功能的范围是否包含 L3 需求？
- [ ] **RTM 一致性**：决策追踪矩阵中的引用是否正确？
- [ ] **优先级一致**：跨层级的优先级定义是否矛盾？

### 正确性验证

- [ ] 需求描述无歧义
- [ ] 验收标准可量化/可验证
- [ ] 决策记录完整（含时间戳和方案类型）
- [ ] 功能模块边界清晰无重叠

## 子Agent独立验证协议

当使用子Agent进行验证时，遵循以下原则确保独立性：

1. **仅传递文档路径列表**，不传递额外上下文
2. **仅传递验证原则**，不告诉预期结果
3. **子Agent输出结构化报告**，由主Agent判断是否需要修复
4. **修复后必须进行额外一轮验证**
