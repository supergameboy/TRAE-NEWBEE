# 需求分析决策框架

## 设计驱动决策原则

需求分析的决策点应**偏向用户需求和业务设计**，而非技术实现。技术选型应在设计确定后，根据实际需求匹配合适的技术方案。

### 决策优先级

```
用户需求/业务价值 > 交互设计 > 数据模型 > 技术架构 > 具体技术选型
```

每个决策点遵循：**先问"要做什么、为什么做"，再问"怎么做、用什么技术"**。

## 三种方案制

每个决策点必须提供 **3 个差异化选项**：

| 方案 | 定位 | 特征 | 适用场景 |
|------|------|------|----------|
| **A - 稳健型** | 保守稳妥 | 成熟方案、低风险、快速交付 | 时间紧迫、团队不熟悉领域 |
| **B - 平衡型（推荐）** | 均衡最优 | 最佳性价比、主流实践 | 大多数场景的默认推荐 |
| **C - 创新型** | 前沿探索 | 新技术/新方法、高潜力但风险较高 | 有充足时间、追求差异化 |

### AskUserQuestion 标准模板

#### 单项选择决策
```markdown
问题：[决策点描述]
选项：
- A: [稳健方案] - [简述优缺点]
- B: [平衡方案-推荐] - [简述优缺点]
- C: [创新方案] - [简述优缺点]
```

#### 范围/优先级排序
```markdown
问题：[需要确认的范围或优先级]
选项：
- [选项1] - [说明]
- [选项2] - [说明]
- [选项3] - [说明]
```

## 各层级决策识别清单

### L0 层级决策点

| 决策类型 | 必须确认的内容 | 方案制应用 |
|----------|---------------|-----------|
| 需求目标确认 | 项目核心目标是什么？为什么做？ | 三种范围方案 |
| 需求边界 | MVP范围 vs 完整功能 vs 未来扩展 | A(最小可行) / B(核心+重要) / C(完整) |
| 目标用户 | 用户画像和核心场景 | 不同用户群体方案 |
| 成功标准 | 如何衡量需求满足程度 | 不同验收粒度 |

### L1 层级决策点

| 决策类型 | 必须确认的内容 | 方案制应用 |
|----------|---------------|-----------|
| 功能模块划分 | 按什么维度拆分模块？ | 按角色/按业务域/按技术层 |
| 模块优先级 | 先做哪些模块？ | 核心流程/完整体验/全功能 |
| 模块间关系 | 模块间依赖和数据流 | 松耦合/标准接口/深度集成 |

### L2 层级决策点

| 决策类型 | 必须确认的内容 | 方案制应用 |
|----------|---------------|-----------|
| 子功能取舍 | 哪些子功能是必须的？ | 最小集合/标准配置/丰富体验 |
| 用户体验路径 | 用户如何完成操作？ | 简捷路径/标准流程/引导式 |
| 数据输入输出 | 输入什么数据？产出什么？ | 手动/自动/智能辅助 |

### L3-L4 层级决策点

| 决策类型 | 必须确认的内容 |
|----------|---------------|
| 具体字段定义 | 每个字段的名称、类型、约束 |
| 业务规则 | 校验规则、计算逻辑、状态流转 |
| 异常处理 | 边界情况、错误提示、降级策略 |
| 交互细节 | UI文案、反馈时机、动画效果 |

## 决策记录格式

每条决策记录到总文档时使用以下格式：

```markdown
| 序号 | 决策点 | 层级 | 方案A(稳健) | 方案B(平衡-推荐) | 方案C(创新) | 用户选择 | 决策时间 | 追溯ID |
|------|--------|------|------------|-----------------|------------|----------|---------|--------|
| 1 | [描述] | L0 | [...] | **[✓]** | [...] | B | HH:MM:SS | REQ-A-001 |
```

## 技术匹配指南

当需求涉及技术相关决策时，先明确需求再搜索匹配技术：

1. **先记录需求**：用户想要什么效果/能力？
2. **WebSearch 搜索**：当前有哪些技术方案能满足此需求？
3. **评估匹配度**：哪个技术与项目现状最匹配？
4. **提供三方案**：基于搜索结果生成三种技术路线
5. **用户选择**：通过 AskUserQuestion 让用户决定

### 常见需求→技术映射示例

| 用户需求方向 | 推荐搜索关键词 | 可能的技术方案 |

| 数据可视化 | "data visualization library {框架}" | D3, ECharts, Recharts |
| 认证授权 | "auth solution 2025-2026" | NextAuth, Clerk, Auth0 |
| AI集成 | "AI SDK integration 2026" | Vercel AI SDK, OpenAI API |
