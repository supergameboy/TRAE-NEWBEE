# 全面审查详细参考

## 全维度评估要点

### 1. 前端评估维度
| 类别 | 详细要点 |
|------|----------|
| 代码规范 | 风格、命名、注释、模块化 |
| 渲染性能 | 加载速度、DOM效率、内存、动画流畅度 |
| 交互逻辑 | 事件处理、状态管理、数据流、反馈 |
| UI还原度 | 设计稿匹配度、像素精度、视觉一致性 |
| 兼容性 | 浏览器兼容、设备适配、响应式 |

### 2. 后端评估维度
| 类别 | 详细要点 |
|------|----------|
| 接口规范 | RESTful设计、API文档、参数验证、响应格式 |
| 业务逻辑 | 正确性、完整性、边界条件、业务规则 |
| 异常处理 | 错误捕获、错误信息、日志、优雅降级 |
| 安全防护 | 输入验证、SQL注入、XSS、认证授权 |

### 3. 数据库评估维度
| 类别 | 详细要点 |
|------|----------|
| SQL规范 | 语法、命名、格式化、注释 |
| 索引优化 | 设计、使用、维护、查询优化 |
| 事务处理 | 边界、隔离级别、并发控制、回滚 |
| 数据一致性 | 约束设计、校验、同步、冗余控制 |

### 4. UI/UX评估维度
| 类别 | 详细要点 |
|------|----------|
| 布局合理性 | 信息架构、空间利用、视觉层次 |
| 视觉效果 | 色彩搭配、字体选择、图标、动效 |
| 用户体验 | 操作流程、反馈机制、错误提示、易用性 |
| 响应式适配 | 多设备适配、断点设计、内容重排 |
