# 自动任务经验总结器

## 功能说明

本技能用于在每次任务执行前读取任务经验，每次任务执行完成后，自动总结任务经验，包括但不限于：
- 任务执行过程中的关键步骤
- 遇到的问题及解决方案
- 优化建议
- 任务流程记录
- 任务类型分类
- 经验类别标注

## 何时使用

**必须在以下情况自动触发：**
- 任何任务执行开始前
- 任何任务执行完成后
- 主Agent标记任务为完成时
- 需要参考任务经验时
- 需要记录任务经验以供后续参考时

## 使用方法

### 触发条件
每次任务执行完成后，主Agent应自动调用此技能。

### 总结内容要求

经验总结必须包含以下元数据：

| 字段 | 说明 | 示例 |
|------|------|------|
| `任务标题` | 简洁明了的任务名称 | "创建用户登录模块" |
| `完成时间` | 任务完成的时间戳 | "2026-03-21 14:30:00" |
| `任务类型` | 任务分类 | "开发" / "测试" / "修复" / "文档" |
| `经验类别` | 经验分类 | "前端" / "后端" / "架构" / "调试" |
| `关键步骤` | 任务执行的主要步骤 | 列表形式 |
| `遇到的问题` | 执行过程中遇到的问题 | 列表形式 |
| `解决方案` | 对应问题的解决方法 | 列表形式 |
| `优化建议` | 未来改进的建议 | 列表形式 |

### 输出格式

经验总结应以标准 Markdown 格式输出，并包含适当的章节结构。

## 经验文件创建

当总结完成后，应将经验总结内容以标准方法创建为新的经验文件，保存路径为：
- `.trae/skills/auto-task-experience-summarizer/experience-{日期}-{序号}/SKILL.md`

其中：
- `{日期}` 格式为 `YYYY-MM-DD`
- `{序号}` 为当天的经验序号，从 01 开始

## 经验索引更新
经验文件创建完毕后，更新索引文件，文件路径为：
- `.trae/skills/auto-task-experience-summarizer/experience-index.md`


经验索引必须包含以下元数据：

| 字段 | 说明 | 示例 |
|------|------|------|
| `任务标题` | 简洁明了的任务名称 | "创建用户登录模块" |
| `完成时间` | 任务完成的时间戳 | "2026-03-21 14:30:00" |
| `任务类型` | 任务分类 | "开发" / "测试" / "修复" / "文档" |
| `经验类别` | 经验分类 | "前端" / "后端" / "架构" / "调试" |
| `遇到的问题` | 执行过程中遇到的问题 | 列表形式 |
| `对应文件` | 经验文件的保存位置 | 相对路径 |


## 注意事项

1. 确保经验总结内容准确反映任务执行情况
2. 包含所有必要的元数据
3. 使用清晰的结构组织内容
4. 避免技术债记录，专注于可复用的经验
5. 遵循项目规范和代码风格
