# Refactor Fractal - 分形式代码重构技能

## 技能概述

本技能采用**分形递归思想**，从重构目标开始，通过多层级、逐步细化的方式制定重构计划并执行。每个决策点、分叉点、需要确认的地方都必须询问用户，每一步重构内容都会保存到文档，防止信息丢失。

## 分形工作流程

### 分形层级定义

| 层级 | 名称 | 说明 | 颗粒度 |
|------|------|------|--------|
| L0 | 重构目标 | 为什么重构、想要达到什么效果 | 目标级 |
| L1 | 范围确定 | 重构哪些模块/文件 | 范围级 |
| L2 | 策略选择 | 用什么重构模式/方法 | 策略级 |
| L3 | 计划制定 | 具体重构步骤 | 计划级 |
| L4 | 执行验证 | 执行重构并验证 | 执行级（可选） |

### 递归重构模式（自相似）

每个层级都遵循相同的重构模式：

```
┌───────────────────────────────────────────────────────────────┐
│  层级 N 重构模式（自相似）                                      │
├───────────────────────────────────────────────────────────────┤
│  1. 层级 N 重构分析 → 2. 识别决策点/分叉点/需要确认点          │
│  ↓                                                              │
│  3. 询问用户确认 → 4. 记录重构内容 → 5. 保存当前重构文档       │
│  ↓                                                              │
│  6. 判断是否深入下一层级（递归决策点）                          │
│  ↓ 是（递归）                                                   │
│  层级 N+1 重构（重复上述模式）                                  │
└───────────────────────────────────────────────────────────────┘
```

## 关键规则

### 主Agent规则
- **必须**严格按照分形层级逐步推进
- **必须**在每个决策点使用 AskUserQuestion 工具询问用户
- **必须**在每一步完成后立即保存文档
- **必须**记录所有用户决策
- **不要**删除代码，而是注释掉
- **不要**自作主张做决策

### 决策点识别规则
以下情况**必须**询问用户：
1. 重构目标确认
2. 重构范围选择
3. 重构策略选择
4. 重构步骤确认
5. 任何有多种可能的情况

### 文档保存
- 保存到：`docs/refactor/refactor-fractal-{YYYYMMDD}-L[0-4].md`

## 何时使用

- 代码需要重构时
- 技术债务需要清理时
- 代码结构需要优化时
- 需要提高代码可维护性时
