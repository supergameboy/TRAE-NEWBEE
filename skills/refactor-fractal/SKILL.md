# Refactor Fractal - 分形式代码重构技能

## 技能概述

本技能采用**分形递归思想** + **横向拆分思想**，从重构目标开始，通过多层级、逐步细化的方式制定重构计划并执行。
- **纵向**：按层级逐步细化（L0 → L1 → L2 → L3 → L4）
- **横向**：每个层级按功能模块/分类拆分，同级并行执行
- **文档**：单一总文档，层级嵌套保存
- **验证**：任务完成后进行正向和反向验证

每个决策点、分叉点、需要确认的地方都必须询问用户，每一步重构内容都会保存到文档，防止信息丢失。

## 分形工作流程

### 分形层级定义

| 层级 | 名称 | 说明 | 颗粒度 | 横向拆分 |
|------|------|------|--------|----------|
| L0 | 重构目标 | 为什么重构、想要达到什么效果 | 目标级 | 无（单独执行） |
| L1 | 功能模块级 | 按功能模块拆分 | 范围级 | 功能模块A、功能模块B、功能模块C... |
| L2 | 子模块级 | 功能模块下的子模块 | 策略级 | 子模块1、子模块2、子模块3... |
| L3 | 文件级 | 具体文件 | 计划级 | 文件1、文件2、文件3... |
| L4 | 代码级 | 具体代码段/函数 | 执行级（可选） | 代码段1、代码段2、代码段3... |

### 递归重构模式（自相似）

每个层级都遵循相同的重构模式：

```
┌───────────────────────────────────────────────────────────────┐
│  层级 N 重构模式（自相似）                                      │
├───────────────────────────────────────────────────────────────┤
│  1. 层级 N 重构分析 → 2. 识别决策点/分叉点/需要确认点          │
│  ↓                                                              │
│  3. 询问用户确认 → 4. 记录重构内容 → 5. 推荐横向拆分方案       │
│  ↓ 询问用户确认（如有）                                         │
│  6. 保存当前内容到总文档 → 7. 判断是否深入下一层级（递归决策点）│
│  ↓ 是（递归）                                                   │
│  层级 N+1 重构（重复上述模式）                                  │
└───────────────────────────────────────────────────────────────┘
```

### 横向并行执行模式

```
L0（单独执行）
    ↓
L1（按功能模块拆分，多个子Agent并行）
    ├─ 功能模块A（子Agent 1）
    ├─ 功能模块B（子Agent 2）
    ├─ 功能模块C（子Agent 3）
    └─ ...
    ↓
总结、检验、修改总文档
    ↓
L2（按子模块拆分，多个子Agent并行）
    ↓
总结、检验、修改总文档
    ↓
...
    ↓
总结、正向+反向验证
```

# 总结与验证

## 正向验证（重构 → 文档）
逐条检查每条重构，确保都有明确的文档依据：
- [ ] 验证项1
- [ ] 验证项2

## 反向验证（文档 → 重构）
遍历所有文档，确保文档中的每个要求都在重构中体现：
- [ ] 验证项1
- [ ] 验证项2

## 正确性验证
检查每个层级的重构方案是否正确：
- [ ] L0 正确性
- [ ] L1 正确性
- [ ] L2 正确性
- [ ] ...

## 一致性验证
确保各层级之间的内容一致：
- [ ] L0 → L1 一致性
- [ ] L1 → L2 一致性
- [ ] ...

### 文档保存时机
- **任务开始时**：创建任务总文档，路径：`docs/refactor/refactor-fractal-{YYYYMMDD}.md`
- **L0 完成后**：写入 L0 内容，询问 L1 横向拆分方案
- **每个 L1 模块完成后**：写入对应 L1 内容，询问 L2 横向拆分方案
- **每个层级全部完成后**：总结、检验、修改总文档
- **任务完成后**：进行正向+反向+正确性+一致性验证，写入验证结果

## 关键规则

### 主Agent规则
- **必须**严格按照分形层级逐步推进，每层级都应用自相似重构模式
- **必须**在每个层级开始时推荐横向拆分方案，使用 AskUserQuestion 询问用户确认
- **必须**调用多个子Agent并行执行同级任务
- **必须**每个层级全部完成后，总结、检验、修改总文档
- **必须**任务完成后进行正向+反向+正确性+一致性验证
- **必须**使用单一总文档，所有层级内容嵌套保存
- **必须**在每个决策点、分叉点、需要确认的地方使用 AskUserQuestion 工具询问用户
- **不要**删除代码，而是注释掉
- **不要**自作主张做决策
- 完成的重构工作写到 `docs/achievement/achievement-{日期}.md` 中

### 决策点识别规则
以下情况**必须**询问用户：
1. 重构目标确认
2. 重构范围选择
3. 重构策略选择
4. 重构步骤确认
5. 横向拆分方案确认
6. 任何有多种可能的情况

### 递归终止条件
- 用户选择不继续深入下一层级
- 用户主动要求停止

## 注意事项

- **每个决策点**都要展示清晰的选项，让用户了解各种可能性
- 每个层级都要推荐清晰的横向拆分方案
- 给予用户充分的选择权，不要预设答案
- 同级任务并行执行，提高效率
- 递归过程中保持上下文连贯性
- **每一步都要保存文档**，防止信息丢失
- 单一总文档，层级嵌套保存，便于追溯
- 记录所有重构过程，便于后续追溯
- 如果有多个可能的重构方向，要询问用户先尝试哪个
- 如果遇到分叉点、需要决策或确认的地方，**必须**使用 AskUserQuestion 工具问用户，**不要**自作主张
