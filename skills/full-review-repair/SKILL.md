# Full Review &amp; Repair 全维度审查修复流程

## 流程概述

本技能用于执行完整的项目代码审查、修复和测试流程，确保代码质量和功能完整性。

## 工作流程

### 简单流程图
```
后端审查（包括数据库）→ 展示审查报告 → 询问修复哪些问题 → 进行后端修复 
→ 再次后端审查 → 展示二次审查报告 → 是否有未修复问题：
  - 是：询问修复哪些问题（回到后端修复）
  - 否：询问是否开始审查前端（包括UI）
→ 前端代码审查 → 展示前端审查报告 → 询问修复哪些问题 → 进行前端修复 
→ 再次前端审查 → 展示二次审查报告 → 是否有未修复问题：
  - 是：询问修复哪些问题（回到前端修复）
  - 否：询问是否开始测试前端操作
→ 使用 frontend-ui-test 技能测试所有前端操作 → 展示测试报告 
→ 询问修复哪些问题 → 进行前端修复 → 再次测试 → ...（循环直到满意）
```

## 全维度评估范围

### 1. 后端
- 接口规范
- 业务逻辑
- 异常处理
- 安全防护
- 代码可读性

### 2. 数据库
- SQL 规范
- 索引优化
- 事务处理
- 数据一致性
- 查询性能

### 3. 前端
- 代码规范
- 渲染性能
- 交互逻辑
- UI 还原度
- 兼容性

### 4. UI
- 布局合理性
- 视觉效果
- 用户体验
- 响应式适配

### 5. 提示词工程（如有）
- 指令精准度
- 逻辑完整性
- 输出可控性
- 场景适配性

## 使用说明

### 前端测试注意事项
- 前端评估需要启动前后端服务器
- 使用 frontend-ui-test 技能进行前端界面操作测试
- 注意浏览器控制台消息
- 注意后台终端消息
- 测试前由主Agent统一启动服务器，测试Agent禁止直接启动服务器

### 关键规则
- 每次审查后向用户展示报告，询问修复哪些问题
- 前后端都修复完毕后，再次进行测试，保证不会有新出现的问题
- 不要删除代码，而是注释掉
- 完成的工作写到 docs/achievement/achievement-{日期}.md 中
- 未完成的工作写到 docs/todo.md 中

### 子Agent规则
- 调用子Agent时，必须明确告诉子Agent的身份
- 子Agent只能申请测试，禁止独自测试（测试Agent除外）
- 在 docs/TestApplication.md 中写需要测试的内容
- 让子Agent复述上述规则

### 主Agent规则
- 主Agent看到子Agent申请测试后，必须根据申请调用测试Agent进行测试
- 在 docs/TestApplication.md 中读需要测试的内容，测试完成后删除内容
- 主Agent复述上述规则
- 测试前，主Agent统一启动服务器，测试Agent禁止直接启动服务器
- 测试完成后，主Agent统一分配任务进行错误修改

## 何时使用

- 项目阶段性审查
- 新功能开发完成后
- 修复bug后
- 用户要求进行全面代码审查和测试时
