# Knowledge Fractal - 分形式知识体系构建技能

## 技能概述

本技能采用**分形递归思想**，从知识主题开始，通过多层级、逐步细化的方式构建知识体系，可用于写帮助文档或总结 achievement。每个决策点、分叉点、需要确认的地方都必须询问用户，每一步知识内容都会保存到文档，防止信息丢失。

## 分形工作流程

### 分形层级定义

| 层级 | 名称 | 说明 | 颗粒度 |
|------|------|------|--------|
| L0 | 知识主题 | 要整理什么知识/主题 | 主题级 |
| L1 | 知识分类 | 分为哪些大类 | 分类级 |
| L2 | 知识结构 | 具体章节/模块 | 结构级 |
| L3 | 知识内容 | 具体内容要点 | 内容级 |
| L4 | 详细展开 | 具体细节、例子、代码 | 细节级（可选） |

### 递归构建模式（自相似）

每个层级都遵循相同的构建模式：

```
┌───────────────────────────────────────────────────────────────┐
│  层级 N 构建模式（自相似）                                      │
├───────────────────────────────────────────────────────────────┤
│  1. 层级 N 知识分析 → 2. 识别决策点/分叉点/需要确认点          │
│  ↓                                                              │
│  3. 询问用户确认 → 4. 记录知识内容 → 5. 保存当前知识文档       │
│  ↓                                                              │
│  6. 判断是否深入下一层级（递归决策点）                          │
│  ↓ 是（递归）                                                   │
│  层级 N+1 构建（重复上述模式）                                  │
└───────────────────────────────────────────────────────────────┘
```

## 关键规则

### 主Agent规则
- **必须**严格按照分形层级逐步推进
- **必须**在每个决策点使用 AskUserQuestion 工具询问用户
- **必须**在每一步完成后立即保存文档
- **必须**记录所有用户决策
- **不要**自作主张做决策

### 决策点识别规则
以下情况**必须**询问用户：
1. 知识主题确认
2. 知识分类选择
3. 知识结构确认
4. 知识内容确认
5. 任何有多种可能的情况

### 文档保存
- 用于帮助文档：保存到 `docs/help/help-fractal-{YYYYMMDD}-L[0-4].md`
- 用于总结 achievement：保存到 `docs/achievement/achievement-{YYYYMMDD}.md`

## 何时使用

- 需要写帮助文档时
- 需要总结 achievement 时
- 需要整理项目知识体系时
- 需要梳理技术栈文档时
- 需要系统化整理信息时
