# Knowledge Fractal - 分形式知识体系构建技能

## 技能概述

本技能采用**分形递归思想** + **横向拆分思想**，从知识主题开始，通过多层级、逐步细化的方式构建知识体系，可用于写帮助文档或总结 achievement。
- **纵向**：按层级逐步细化（L0 → L1 → L2 → L3 → L4）
- **横向**：每个层级按功能模块/分类拆分，同级并行执行
- **文档**：单一总文档，层级嵌套保存
- **验证**：任务完成后进行正向和反向验证

每个决策点、分叉点、需要确认的地方都必须询问用户，每一步知识内容都会保存到文档，防止信息丢失。

## 分形工作流程

### 分形层级定义

| 层级 | 名称 | 说明 | 颗粒度 | 横向拆分 |
|------|------|------|--------|----------|
| L0 | 知识主题 | 要整理什么知识/主题 | 主题级 | 无（单独执行） |
| L1 | 功能模块级 | 按功能模块拆分 | 模块级 | 功能模块A、功能模块B、功能模块C... |
| L2 | 子模块级 | 功能模块下的子模块 | 子模块级 | 子模块1、子模块2、子模块3... |
| L3 | 知识点级 | 具体知识点 | 内容级 | 知识点1、知识点2、知识点3... |
| L4 | 细节级 | 具体细节、例子、代码 | 细节级（可选） | 细节1、细节2、细节3... |

### 递归构建模式（自相似）

每个层级都遵循相同的构建模式：

```
┌───────────────────────────────────────────────────────────────┐
│  层级 N 构建模式（自相似）                                      │
├───────────────────────────────────────────────────────────────┤
│  1. 层级 N 知识分析 → 2. 识别决策点/分叉点/需要确认点          │
│  ↓                                                              │
│  3. 询问用户确认 → 4. 记录知识内容 → 5. 推荐横向拆分方案       │
│  ↓ 询问用户确认（如有）                                         │
│  6. 保存当前内容到总文档 → 7. 判断是否深入下一层级（递归决策点）│
│  ↓ 是（递归）                                                   │
│  层级 N+1 构建（重复上述模式）                                  │
└───────────────────────────────────────────────────────────────┘
```

### 横向并行执行模式

```
L0（单独执行）
    ↓
L1（按功能模块拆分，多个子Agent并行）
    ├─ 功能模块A（子Agent 1）
    ├─ 功能模块B（子Agent 2）
    ├─ 功能模块C（子Agent 3）
    └─ ...
    ↓
总结、检验、修改总文档
    ↓
L2（按子模块拆分，多个子Agent并行）
    ↓
总结、检验、修改总文档
    ↓
...
    ↓
总结、正向+反向验证
```

### 文档保存时机
- **任务开始时**：创建任务总文档，路径：`docs/help/knowledge-fractal-{YYYYMMDD}.md` 或 `docs/achievement/achievement-{YYYYMMDD}.md`

- **每个 L1 模块完成后**：写入对应 L1 内容，询问 L2 横向拆分方案
- **每个层级全部完成后**：总结、检验、修改总文档
- **任务完成后**：进行正向+反向+正确性+一致性验证，写入验证结果

## 详细步骤

### 第一阶段：L0 知识主题（主题级）- 单独执行

#### 1.1 创建任务总文档
创建单一总文档，路径根据用途选择：
- 用于帮助文档：`docs/help/knowledge-fractal-{YYYYMMDD}.md`
- 用于总结 achievement：`docs/achievement/achievement-{YYYYMMDD}.md`

#### 1.2 收集知识主题
- 请用户描述要整理的知识/主题
- 记录用户的原始描述
- 确认理解是否正确

#### 1.3 识别 L0 决策点
- 知识主题确认
- 构建目标确认
- 目标读者确认

#### 1.4 询问用户确认
使用 AskUserQuestion 工具询问用户关于 L0 决策点的信息。

#### 1.5 推荐 L1 横向拆分方案
向用户推荐 L1 层级的横向拆分方案（按功能模块）：
| 序号 | L1 功能模块 | 说明 | 建议优先级 |
|------|------------|------|-----------|
| 1 | 功能模块A | [说明] | 高 |
| 2 | 功能模块B | [说明] | 中 |
| 3 | 功能模块C | [说明] | 中 |

**重要**：必须使用 AskUserQuestion 工具询问用户确认/调整拆分方案！

#### 1.6 写入 L0 内容到总文档
将 L0 所有内容写入总文档。

#### 1.7 询问是否进入 L1 知识分类

---

### 第二阶段：L1 功能模块级知识（模块级）- 同级并行

#### 2.1 L1 横向并行执行
根据用户确认的 L1 拆分方案，调用多个子Agent并行执行：
- 子Agent 1：功能模块A
- 子Agent 2：功能模块B
- 子Agent 3：功能模块C
- ...

#### 2.2 每个 L1 模块的构建流程（自相似）

```
┌─────────────────────────────────────────────────────┐
│ L1 模块构建（应用自相似模式）                        │
├─────────────────────────────────────────────────────┤
│ 1. L1 知识分析 → 2. 识别决策点 → 3. 询问用户         │
│ ↓                                                     │
│ 4. 记录知识内容 → 5. 推荐 L2 拆分方案 → 6. 询问用户 │
│ ↓                                                     │
│ 7. 写入总文档 → 8. 是否深入 L2？                      │
└─────────────────────────────────────────────────────┘
```

#### 2.3 L1 知识内容
- 功能模块概述
- 主要知识点识别
- 知识结构初步确认

#### 2.4 L1 决策点
| 决策点 | 说明 |
|--------|------|
| 知识清单 | 列出所有识别到的知识 |
| 知识类型 | 每个知识的类型/角色 |
| 关键知识 | 哪些知识是核心/关键知识 |
| 是否有遗漏知识 | 是否需要补充知识 |

**重要**：以上决策点必须使用 AskUserQuestion 工具询问用户！

#### 2.5 推荐 L2 横向拆分方案
每个 L1 模块完成后，向用户推荐 L2 层级的横向拆分方案（按子模块）。

**重要**：必须使用 AskUserQuestion 工具询问用户确认/调整拆分方案！

#### 2.6 汇总所有 L1 模块结果
主Agent整合所有子Agent的 L1 结果，总结、检验、修改总文档。

#### 2.7 进入 L2 阶段（或跳过）

---

### 第三阶段：L2 子模块级知识（子模块级）- 同级并行

#### 3.1 L2 横向并行执行
对于每个 L1 模块下的 L2 子模块，调用多个子Agent并行执行。

#### 3.2 L2 构建流程（继续自相似模式）
同 L1 模式，推荐 L3 横向拆分方案（按知识点）。

#### 3.3 汇总所有 L2 子模块结果
主Agent整合所有子Agent的 L2 结果，总结、检验、修改总文档。

---

### 第四阶段：L3 知识点级知识（内容级）- 同级并行

#### 4.1 L3 横向并行执行
对于每个 L2 子模块下的 L3 知识点，调用多个子Agent并行执行。

#### 4.2 L3 构建流程（继续自相似模式）
同 L1 模式，可选推荐 L4 横向拆分方案（按细节）。

#### 4.3 汇总所有 L3 知识点结果
主Agent整合所有子Agent的 L3 结果，总结、检验、修改总文档。

---

### 第五阶段：L4 细节级知识（细节级）- 可选深度递归

#### 5.1 L4 横向并行执行（可选）
对于每个 L3 知识点下的 L4 细节，调用多个子Agent并行执行。

#### 5.2 L4 构建流程（继续自相似模式）
同 L1 模式，递归终止。

#### 5.3 汇总所有 L4 细节结果
主Agent整合所有子Agent的 L4 结果，总结、检验、修改总文档。

---

### 第六阶段：总结与验证

#### 6.1 正向验证（知识 → 文档）
逐条检查每条知识，确保都有明确的文档依据：
- 文档名称
- 章节/行号
- 具体引用内容

#### 6.2 反向验证（文档 → 知识）
遍历所有文档，确保文档中的每个信息都在知识中体现：
- 建立文档内容清单
- 与知识列表逐项比对
- 标记遗漏项
- 补充缺失知识

#### 6.3 正确性验证
检查每个层级的知识整理是否正确：
- L0 正确性
- L1 正确性
- L2 正确性
- ...

#### 6.4 一致性验证
确保各层级之间的内容一致：
- L0 → L1 一致性
- L1 → L2 一致性
- ...

#### 6.5 生成知识体系总结
汇总所有层级的知识内容，生成知识体系总结。

#### 6.6 写入验证结果到总文档
将所有验证结果、知识体系总结写入总文档。

## 关键规则

### 主Agent规则
- **必须**严格按照分形层级逐步推进，每层级都应用自相似构建模式
- **必须**在每个层级开始时推荐横向拆分方案，使用 AskUserQuestion 询问用户确认
- **必须**调用多个子Agent并行执行同级任务
- **必须**每个层级全部完成后，总结、检验、修改总文档
- **必须**任务完成后进行正向+反向+正确性+一致性验证
- **必须**使用单一总文档，所有层级内容嵌套保存
- **必须**在每个决策点、分叉点、需要确认的地方使用 AskUserQuestion 工具询问用户
- **不要**自作主张做决策，所有知识整理都必须询问用户确认
- **不要**跳过层级，必须按照 L0→L1→L2→L3→L4 的顺序推进
- 完成的知识整理工作写到 docs/achievement/achievement-{日期}.md 中

### 决策点识别规则
以下情况**必须**询问用户：
1. 知识主题确认
2. 知识分类选择
3. 知识结构确认
4. 知识内容确认
5. 横向拆分方案确认
6. 任何有多种可能的情况
7. 任何不确定用户想法的情况
8. 任何需要用户确认的知识整理方向

### 递归终止条件
- 用户选择不继续深入下一层级
- 已到达 L4 细节级（最细颗粒度）
- 用户主动要求停止
- 知识已经整理到足够详细程度，无需继续深入
- 用户已经确认所有知识内容

## 注意事项

- **每个决策点**都要展示清晰的选项，让用户了解各种可能性
- 每个层级都要推荐清晰的横向拆分方案
- 给予用户充分的选择权，不要预设答案
- 同级任务并行执行，提高效率
- 递归过程中保持上下文连贯性
- **每一步都要保存文档**，防止信息丢失
- 单一总文档，层级嵌套保存，便于追溯
- 记录所有知识整理过程，便于后续追溯
- 如果有多个可能的知识整理方向，要询问用户先尝试哪个
- 如果遇到分叉点、需要决策或确认的地方，**必须**使用 AskUserQuestion 工具问用户，**不要**自作主张
