# 知识体系构建详细参考

## 文档模板结构

```markdown
# 分形式知识体系 - {主题名称} - {YYYYMMDD}

## 任务信息
- 开始时间：{时间}
- 知识主题：{主题描述}
- 构建目标：{目标}

---
# L0 - 知识主题定义（决策记录 + 横向拆分方案）
# L1 - 知识领域级（嵌套 L2→L3→L4）
# 总结与验证
# 知识体系总览 + 知识地图
```

**保存路径**：`docs/knowledge/knowledge-{YYYYMMDD}.md`

## 各层级详细内容

### L0 知识主题定义 — 必须确认的信息

| 信息类别 | 具体问题 |
|----------|----------|
| 知识范围 | 要构建什么知识体系？涵盖哪些领域？ |
| 目标读者 | 谁会使用这些知识？需要什么程度的细节？ |
| 使用场景 | 在什么情况下会被查阅？ |

### L1-L4 逐层细化目标

| 层级 | 定位 | 内容类型 | 示例 |
|------|------|----------|------|
| L0 | 主题定义 | 知识边界、目标、读者 | "项目前端架构知识" |
| L1 | 领域级 | 大类知识分区 | 组件库/状态管理/路由/样式系统 |
| L2 | 分类级 | 领域下的子分类 | useState/useReducer/Context/Zustand |
| L3 | 条目级 | 具体知识点 | 每个hook的用法、参数、示例 |
| L4 | 细节级 | 补充说明、代码片段、注意事项 | 完整代码示例+最佳实践 |

## 知识输出格式规范

### Markdown格式要求

- 使用标准Markdown语法
- 代码块标注语言类型
- 表格用于结构化数据展示
- Mermaid图用于可视化关系
- 层级使用标题层级对应（L0=#, L1=##, L2=###...）

### 索引与导航

- L0层提供知识地图（目录索引）
- 每个条目有唯一ID便于引用
- 相关知识点之间建立交叉引用链接
