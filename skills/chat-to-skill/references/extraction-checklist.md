# extraction-checklist

## 聊天输入检查

开始前检查：

- [ ] 输入是单文件、目录，还是多文件集合
- [ ] 能识别至少一个完整问题到结果闭环
- [ ] 能看到稳定流程，而不只是一次性问题
- [ ] 用户已明确脱敏级别
- [ ] 用户希望输出完整技能包，而不是普通总结

如果任一项不满足，先补充材料或重新定义范围。

## 脱敏检查

在中间产物和最终产物中检查：

- [ ] 没有真实项目名
- [ ] 没有真实仓库名
- [ ] 没有真实本地绝对路径
- [ ] 没有真实用户名目录
- [ ] 没有真实端口号
- [ ] 没有真实文档标题
- [ ] 没有真实品牌名、客户名、业务名
- [ ] 没有可回溯到原聊天的唯一性实体

如果发现残留，先修正，再继续下游生成。

## 中间 outline 检查

- [ ] 已提取任务目标
- [ ] 已提取触发短语
- [ ] 已提取稳定步骤
- [ ] 已提取决策点
- [ ] 已提取验证步骤
- [ ] 已提取常见失误
- [ ] 已提取协作接口
- [ ] 已去除一次性噪音

## 最终技能包检查

- [ ] `SKILL.md` 存在
- [ ] `references/` 至少包含规则文档
- [ ] `examples/` 使用通用示例
- [ ] `scripts/` 包含最小可运行脚本
- [ ] `SKILL.md` 引用了 supporting files
- [ ] frontmatter 使用第三人称
- [ ] 正文使用指令式表达
- [ ] 触发词明确且具体

## 示例检查

- [ ] 示例对话是重写版本
- [ ] 示例输出不含真实项目线索
- [ ] 示例能表达输入到技能结构的映射关系
- [ ] 示例长度足够说明问题，但不过度复制真实材料

## 脚本检查

- [ ] 支持文件输入
- [ ] 支持目录输入
- [ ] 能识别基本 User/Assistant 轮次
- [ ] 能输出结构化 outline
- [ ] 不把原文直接转换为最终 `SKILL.md`

## 总览更新检查

- [ ] `skills/SKILL-SYSTEM-OVERVIEW.md` 已注册新技能
- [ ] 技能总数已更新
- [ ] 定位与分类一致
- [ ] 工作流图或说明已体现新技能角色
