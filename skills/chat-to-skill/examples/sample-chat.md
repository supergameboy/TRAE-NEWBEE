# sample-chat

> 这是一个通用化、不可追溯的示例。仅用于展示“聊天记录如何被提炼为技能”，不对应任何真实项目。

## User

我们有一批历史聊天记录，里面包含问题分析、方案确认、代码修改和验证过程。

想把这些聊天整理成一个独立技能，后续遇到类似任务时可以直接触发复用。

要求先脱敏，不能从技能内容反推回原项目。

## Assistant

先读取聊天源，识别是否存在稳定流程和共性决策点。

再执行严格脱敏，删除真实路径、项目名、文档标题和唯一性实体。

最后分两阶段输出：先形成中间 outline，再组装完整技能包。

## User

不要把真实聊天原文放进 examples。

## Assistant

改为重写后的通用示例，仅保留结构，不保留原实体。

## User

技能包里需要包含：

- `SKILL.md`
- 两份参考文档
- 一个通用示例输入
- 一个通用示例输出
- 一个最小抽取脚本

## Assistant

将技能拆成以下职责：

1. `SKILL.md` 承载主流程和触发词
2. `references/` 承载抽取规则和检查清单
3. `examples/` 承载脱敏后的示例
4. `scripts/` 承载 outline 抽取逻辑

## User

还要更新技能系统总览。

## Assistant

在技能创建完成后，同步更新系统总览中的技能总数、分类和定位说明。
