# 任务规划详细参考

## 文档模板详细说明

### 总规划文档模板 (`plan-master.md`) 章节结构：

- **任务信息**：项目名称、开始时间、规划版本
- **全部任务展示**：Mermaid总览图 + 任务清单表格
- **任务之间联系**：依赖关系图(Mermaid) + 依赖表
- **全部参考文档**：输入文档索引
- **总验收标准**：项目级DoD
- **L0 项目目标**：
  - 文档概览（含分批次读取策略）
  - L0 决策记录（含时间戳+方案类型）
  - 阶段划分方案
- **阶段概览**：各阶段状态摘要（进行中/已完成/未开始）
- **整体总结与验证**：正向/反向/一致性验证结果

### 阶段文档模板 (`plan-phase.md`) 章节结构：

- **阶段信息**：阶段名称、起止时间、状态
- **当前阶段全部任务**：任务树(Mermaid) + 清单表格
- **任务执行流程顺序图**：Mermaid顺序图
- **单个任务的实现规划**：
  - 描述 + 流程图(Mermaid) + 要点 + 验收标准(DoD)
  - 单个任务的任务来源追溯
- **当前阶段任务之间联系**：依赖关系
- **与下一阶段的对接信息**：
  - 对接流程图 + 接口定义 + 数据传递格式 + 交互衔接说明
- **L1/L2/L3/L4 各层级嵌套结构**
- **阶段总结与验证**

## 正反合检验机制详解

### 常规检验机制（文档 ≤10个且 ≤12000行）

1. 调用子Agent执行正向验证（所有任务都有文档依据）
2. 调用子Agent执行反向验证（所有文档要求都有任务去执行）
3. 调用子Agent执行一致性验证（各层级内容一致）
4. 整理所有子Agent的验证报告
5. 使用 AskUserQuestion 询问用户确认修正方案
6. 修正后再次验证（如有需要）

### 分批检验机制（文档 >10个或 >12000行）

#### 完全图多元组合验证方法

**第一步：文档分组（3-5组）**
- A组：通用文档/底层文档（架构、规范、工具类）
- B组：核心业务文档（用户、订单、支付等）
- C组：辅助功能文档（通知、日志、统计等）
- D组：前端文档（UI组件、页面、交互等）
- E组：其他文档（部署、运维、测试等）

**第二步：完全图组合验证**
- 以 n 组为例，共 C(n,2) 个两两验证组合
- 每个组合调用一个独立子Agent
- 输出使用 `verification-report-pair.md` 模板

**第三步：整理报告**
- 主Agent汇总所有子Agent报告
- 输出使用 `verification-summary.md` 模板

**第四至六步**：用户确认修正 → 再次验证 → 循环直到通过

## 分批次读取策略详解

### 默认批次划分

| 批次 | 内容 | 说明 |
|------|------|------|
| 批次1 | 参考文档索引、项目架构说明 | 全局视角 |
| 批次2 | 核心功能设计文档（按模块分组） | 业务核心 |
| 批次3 | 技术实现文档（前端/后端/数据库） | 技术细节 |
| 批次4 | 其他辅助文档 | 补充信息 |

### 读取管理规则

- 每批次读取完成后立即记录到 `batch-read-template.md` 格式
- 记录包含：批次号、文件列表、关键发现、待处理项
- L0 阶段必须完成全部分批次读取后才进入 L1 规划
