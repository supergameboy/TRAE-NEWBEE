# 问题排查详细参考

## 文档模板结构

```markdown
# 分形式问题排查 - {问题描述} - {YYYYMMDD}

## 任务信息
- 开始时间：{时间}
- 问题描述：{简述}
- 排查目标：{目标}

---
# L0 - 现象描述（决策记录 + 横向拆分方案）
# L1 - 领域判断（前端/后端/数据库/网络/其他）
# L2 - 模块定位（嵌套在L1下）
# L3 - 文件定位（嵌套在L2下）
# L4 - 代码定位（可选，嵌套在L3下）
# L5 - 根因分析（可选）
# 总结与验证 + 根因总结 + 修复建议 + 预防措施
```

**保存路径**：`docs/debug/bug-hunt-{YYYYMMDD}.md`

## 各层级详细内容

### L0 现象描述 — 必须收集的信息

| 信息类别 | 具体问题 |
|----------|----------|
| 时间信息 | 问题什么时候出现的？出现频率？ |
| 复现信息 | 是否稳定复现？偶发？需要特定条件？ |
| 操作信息 | 问题出现前做了什么操作？ |
| 预期行为 | 正常情况下应该是什么表现？ |

### L1 领域判断 — 五大领域

| 领域 | 表现特征 | 优先级 |
|------|----------|--------|
| 前端 | 页面显示异常、交互问题、控制台错误 | 高 |
| 后端 | API响应异常、服务器日志错误、错误码 | 高 |
| 数据库 | 数据异常、查询错误、连接问题 | 中 |
| 网络 | 请求失败、超时、跨域问题 | 中 |
| 其他 | 环境/配置/依赖问题 | 低 |

### L2-L4 逐层细化目标

| 层级 | 定位目标 | 决策重点 |
|------|----------|----------|
| L2 模块级 | 哪个功能模块有问题？ | 相关模块清单、可疑模块、排查顺序 |
| L3 文件级 | 哪个文件有问题？ | 相关文件、可疑文件、排查顺序 |
| L4 代码级 | 哪段代码有问题？ | 相关代码段、可疑代码、排查顺序 |

### L5 根因分析 — 五问法

1. **根本原因是什么？** — 直接原因 vs 根本原因
2. **为什么会出现这个问题？** — 触发条件分析
3. **什么条件触发了问题？** — 前置条件链
4. **如何避免类似问题再次发生？** — 预防措施
5. **是否有系统性改进空间？** — 从单点修复到系统优化

## 常见BUG模式与搜索方向

当遇到以下情况时，使用 WebSearch 搜索最新解决方案：

| BUG模式 | 搜索关键词示例 |
|---------|---------------|
| 内存泄漏 | "{语言} memory leak debugging 2025-2026" |
| 并发竞态 | "{框架} race condition fix" |
| 性能瓶颈 | "{技术栈} performance profiling tools" |
| 安全漏洞 | "{框架} security vulnerability {版本}" |
| 兼容性问题 | "{库名} compatibility issue {环境}" |
