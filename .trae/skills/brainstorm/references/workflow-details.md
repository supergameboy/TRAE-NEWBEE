# 头脑风暴详细工作流程

## 功能区域划分标准

### 后端架构
- 路由定义与配置、服务层实现、数据访问层/仓库模式
- 中间件配置与实现、数据库连接与操作、API接口设计与实现

### 前端架构
- 组件层次结构、页面布局与实现、路由配置与导航
- 状态管理策略与实现、API服务封装与调用、前端构建流程

### 数据库
- 表结构设计、索引策略、数据迁移脚本、常用查询语句
- 事务管理、数据备份与恢复机制

### 配置与工具
- 环境配置文件、构建工具配置、依赖管理
- 代码质量工具、部署脚本、CI/CD配置

### 项目其他相关
- 项目文档、API文档、代码注释规范
- 用户反馈记录、版本历史、已知问题清单

**注：根据项目实际需求动态添加功能区域**

## Search Agent 探索报告格式

每个 Search Agent 返回的报告需包含：

1. **核心功能概述**（200-300字）：该区域的功能与实现概述
2. **关键文件列表**：文件路径、文件名及简要作用说明
3. **架构关系图**：现有架构关系（如无可提供结构化描述）
4. **任务关联分析**：
   - 任务目标与该功能区域的直接关联点
   - 该功能区域与其他区域的交互方式和依赖关系
   - 与系统其他核心组件的接口定义和数据流向

## 专业Agent方案格式

每个专业Agent提出的方案必须包含：

1. **方案名称**：简明扼要的标题（快速/平衡/长远）
2. **详细实现步骤**：分步骤的操作指南
3. **技术选型建议**：推荐的技术栈和工具
4. **潜在风险评估**：可能的风险点和缓解措施
5. **预计工作量评估**：时间和复杂度预估

**约束**：
- 严格禁止在方案阶段直接修改任何代码
- 每个任务提出恰好3个方案（快速解决/均衡/长远计划）

## 方案整合评估维度

主Agent对每个方案的评估包含：

| 维度 | 说明 | 权重 |
|------|------|------|
| 技术可行性 | 方案在技术上是否可实现 | 高 |
| 创新性 | 方案的技术优点和创新点 | 中 |
| 局限性 | 潜在缺点和局限性 | 高 |
| 风险评估 | 实施风险和缓解难度 | 高 |
| 目标契合度 | 与原始目标的匹配程度 | 高 |
| 成本效益 | 投入产出比 | 中 |
| 推荐指数 | 综合评分(0-10) | — |

## 子Agent身份确认协议

调用子Agent时，必须在初始指令中要求子Agent确认：

> "我只能提出方案建议，不能独自进行实际测试或执行操作"

确保子Agent明确自身权限边界。
