# 重构详细参考

## 文档模板结构

```markdown
# 分形式代码重构 - {项目名称} - {YYYYMMDD}
## 任务信息（开始时间、项目名、重构目标）
# L0-L4 各层级嵌套内容
# 总结与验证（正向+反向+正确性+一致性）
# 重构总结 + 重构计划
```

**保存路径**：`docs/refactor/refactor-fractal-{YYYYMMDD}.md`

## 各层级详细重构内容

### L0 重构目标 — 必须确认的信息

| 信息类别 | 具体问题 |
|----------|----------|
| 重构动机 | 为什么需要重构？技术债务？性能？可维护性？ |
| 重构目标 | 想要达到什么效果？具体指标？ |
| 约束条件 | 时间限制？兼容性要求？不能破坏什么？ |

### L1-L3 逐层细化目标

| 层级 | 定位目标 | 决策重点 |
|------|----------|----------|
| L1 | 哪些功能模块需要重构？ | 模块清单、优先级、依赖关系 |
| L2 | 模块下哪些子模块需重构？ | 子模块范围、影响面评估 |
| L3 | 具体哪些文件需要修改？ | 文件清单、修改类型分类 |
| L4 | 哪段代码需要重写？（可选） | 具体函数/类/代码段 |

## 重构策略选项参考

| 重构类型 | 搜索关键词示例 |
|----------|---------------|
| 架构重构 | "monolith to microservices migration {年份}" |
| 代码清理 | "code cleanup best practices {语言}" |
